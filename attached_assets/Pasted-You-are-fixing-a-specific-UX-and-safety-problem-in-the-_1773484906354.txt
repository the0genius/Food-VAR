Your job:

1. user-specific allergen match
2. product-level allergen information
3. inferred or possible allergen risk
4. generic medical disclaimer

Do not change the core design language. Keep the current visual style, spacing, and overall layout. Only change the logic, labels, color semantics, and UI wording needed to remove the contradiction.

GOAL STATE
==================================================

The result screen must communicate these states clearly:

STATE A: hard user-specific allergen alert
- This means the user's saved allergies match declared or verified allergens
- Show red
- Strong copy like:
  - "Allergen Alert"
  - "Contains allergens that match your profile"
- Use `isAllergenAlert === true`
- Use `matchedAllergens`

STATE B: no allergen match for this user, but product still contains allergens
- This means the product contains allergens, but not ones saved in this user's profile
- Do NOT show red danger styling
- Show neutral or amber styling
- Copy should be profile-aware, for example:
  - "No match with your saved allergies"
  - "Product contains: gluten, soy"
- Make it clear this is product information, not a personal alert

STATE C: inferred or possible allergen risk
- This means allergens were inferred from ingredients or lower-confidence data
- Do NOT show red danger styling
- Show amber or warning styling
- Copy examples:
  - "Possible allergen risk from ingredients"
  - "May contain ingredients related to: milk, soy"
- Use `inferredAllergenWarnings`
- Make clear this is not a confirmed hard allergen alert

STATE D: no relevant allergen signal
- No hard match
- No inferred warning
- No important product allergen message for the user
- Show either nothing, or a simple neutral-safe informational row
- Avoid making it feel like a medical guarantee
- Never say "safe"
